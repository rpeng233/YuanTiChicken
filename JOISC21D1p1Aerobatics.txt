For three consecutive points (a,b,c), compute the angle at (b) using

[
\operatorname{atan2}\left(
|(a-b)\times(c-b)|,,
(a-b)\cdot(c-b)
\right).
]

Small Case

For (N=15), use bitmask dynamic programming.

Let

[
dp[S][a][b]
]

be the maximum possible minimum angle of a path that visits exactly the points in (S) and ends with (a,b).

To append an unused point (c),

\max\left(
dp[S\cup{c}][b][c],
\min(dp[S][a][b],\angle abc)
\right).
]

Store parent pointers to reconstruct the optimal path.

Complexity:

[
O(2^N N^3).
]

Large Cases

Use heuristic optimization.

1. Greedy Initialization

Choose two starting points, then repeatedly append the unused point (x) maximizing the new angle

[
\angle p_{k-1}p_kx.
]

Try multiple starting pairs and keep the best route.

2. 2-opt Local Search

Choose two positions and reverse the segment between them.

A segment reversal changes only the angles near its two endpoints; all internal angles remain unchanged. Therefore, each move can be evaluated using only (O(1)) angle computations.

3. Smooth Objective Function

Optimizing only the minimum angle causes many ties. Instead, choose a target angle (\theta) and score each angle (x) by

[
f(x)=
\begin{cases}
x-\theta, & x<\theta,\
C, & x\ge\theta.
\end{cases}
]

Maximize

[
F=\sum_i f(A_i).
]

This encourages all angles to reach the target rather than improving only the current worst angle.

4. Simulated Annealing

Accept improving 2-opt moves. Occasionally accept worsening moves with probability

[
\exp\left(\frac{\Delta F}{T}\right),
]

where (T) gradually decreases.

Run several independent searches with different initial routes, target angles, and random seeds.

Optimization

Represent the route by storing the two neighbors of every point. A 2-opt move then removes two edges and inserts two new edges in (O(1)) time.

Temporary cycle components may be allowed during the search. At the end, reconnect the components to obtain one Hamiltonian path.

Overall Complexity

For (I) local-search iterations:

[
O(N^2+I)
]

after using constant-time edge updates.

The final answer is the best route found over all restarts.
