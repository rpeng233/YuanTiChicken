Use Dijkstra on a layered state graph.

Let `dist[u][p]` be the minimum cost to be at node `u` with a usable parameter value `p`, where only `1 <= p <= d[u]` states are stored. The total number of states is

\[
\sum d[u] = m.
\]

For each state `(u, p)`:

- If `p > 1`, relax `(u, p-1)` with cost `w[p]`.
- If `p < d[u]`, relax `(u, p+1)` with cost `v[p]`.
- Let the `p`-th outgoing edge of `u` go to `to` with cost `c`.
  - Update `ans[to]` with `dist[u][p] + c`.
  - If `d[to] > 0`:
    - If `p <= d[to]`, relax `(to, p)` with cost `c`.
    - Otherwise relax `(to, d[to])` with extra cost for decreasing `p` down to `d[to]`.

Precompute prefix sums of the decreasing costs so that

\[
\text{cost}(p \to q)=\sum_{i=q+1}^{p} w_i
\]

is obtained in \(O(1)\).

The key observation is that states with `p > d[u]` never need to be stored: from such a state no outgoing edge can be used, so the parameter must first be decreased into the valid range. It is enough to jump directly to `d[u]`, and further decreases are handled by normal adjacent transitions.

Initialize `dist[1][1] = 0` if valid, and `ans[1] = 0`. Run Dijkstra over the implicit graph. There are \(O(m)\) states and transitions, so the complexity is

\[
O(m \log m)
\]

with \(O(n+m)\) memory.
