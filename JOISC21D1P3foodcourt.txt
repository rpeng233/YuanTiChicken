# JOISC 2021 Food Court — Solution Summary

Process shops from left to right. For each range event on shops $[L,R]$, activate it at $L$ and deactivate it at $R+1$. While processing shop $a$, maintain a segment tree over event times containing exactly the events that affect shop $a$.

## Queue length

Represent each operation as a function

$$
f(z)=\max(z+x,y).
$$

* Adding $K$ customers corresponds to $(x,y)=(K,-\infty)$.
* Removing $K$ customers corresponds to $(x,y)=(-K,0)$.

These functions are closed under composition:

$$
(x_1,y_1)\circ(x_2,y_2)
=======================

\left(x_1+x_2,\max(y_1+x_2,y_2)\right),
$$

with the order chosen to match chronological processing.

Store the composition of active events in each segment-tree node. For a query at time $t$, applying the composition of events in $[1,t]$ to $0$ gives the current queue length $S$.

## Finding the requested customer

Also store, for every active insertion event, the number of inserted customers. Let

$$
T=\text{total number of customers ever inserted before time }t.
$$

If the query asks for the $B$-th customer and $S<B$, output $0$.

Otherwise, $T-S$ customers have already left from the front, so the desired customer is the

$$
P=T-S+B
$$

-th customer among all customers ever inserted.

Use the insertion-count segment tree to find the first event-time prefix whose total inserted count is at least $P$. The insertion event at that position contains the desired customer, so output its group identifier.

## Complexity

Each event is activated and deactivated once, and every segment-tree operation costs $O(\log Q)$.

$$
\boxed{O((N+Q)\log Q)\text{ time}}
$$

$$
\boxed{O(N+Q)\text{ memory}}
$$
