For a fixed initial length \(k\), split the remaining sequence into draw blocks. For block \(i\), let \(a_i,b_i\) be the numbers of \(0,1\), and define

\[
l_i=1-b_i,\qquad r_i=a_i-1.
\]

Let \(h\) be the number of \(1\)'s in the initial prefix, and let \(c_i\in\{0,1\}\) indicate whether the special replacement is used in block \(i\). With \(s_i=\sum_{j\le i}c_j\), feasibility becomes a system of prefix constraints:

\[
s_i\ge L_i-h,\qquad s_i\le k-h+R_i,
\]

plus the cooldown constraint

\[
s_y-s_x\le 1\quad (0\le x<y,\ y-x\le 3).
\]

This difference-constraint system can be eliminated. For the suffix after the initial prefix, define

\[
A_i=3l_i-1,\qquad B_i=-3r_i.
\]

Then \(k\) is feasible iff all four inequalities hold:

\[
3h\ge \max\operatorname{pref}(A)-2,
\]

\[
3(k-h)\ge \max\operatorname{pref}(B)-2,
\]

\[
3k\ge \max\operatorname{subarray}(A)-2,
\]

\[
3k\ge \max\operatorname{subarray}(B)-2.
\]

Thus the problem reduces to maintaining prefix maxima, suffix maxima, subarray maxima, and sums for two arrays under point updates.

The block decomposition depends only on the parity of \(k\). Build two segment trees:

- one for blocks \([1],[2,3],[4,5],\ldots\);
- one for blocks \([1,2],[3,4],[5,6],\ldots\).

Each candidate \(k\) corresponds to taking a prefix of blocks in one of these trees, so the needed data is exactly the aggregate of the remaining suffix. Each node stores the sums of \(A,B\), maximum prefix/suffix/subarray sums for both, and the number of \(1\)'s in the interval. Merging is the standard maximum-subarray merge.

A flip affects exactly one block in each parity tree, so update both segment trees in \(O(\log n)\). To find the answer, either binary-search the first feasible block endpoint using suffix queries, giving \(O(\log^2 n)\) per operation, or descend the tree while carrying the already chosen right suffix aggregate, giving \(O(\log n)\). Compute the smallest feasible endpoint in both parity trees and output their minimum.

Total complexity is \(O(n+q\log n)\) with tree descent, or \(O(n+q\log^2 n)\) with the simpler binary-search version.
