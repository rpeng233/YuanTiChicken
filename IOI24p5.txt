Define layers by `min(row, col)`. The key observation is:

- Layer `0` is known.
- Layer `1` has no adjacent `1`s.
- Layer `2` has no adjacent `1`s and no three adjacent `0`s.
- Every later layer is just a diagonal copy inward:  
  `A[i][j] = A[i-1][j-1]`.

Thus any cell with `min(i,j) >= 2` can be projected diagonally back to layer `2`, so the interior is determined by one flattened sequence.

Compute layers `1` and `2` in `O(N)`. Store layer `2` as a sequence indexed by diagonal, and build prefix sums of:

- values,
- index × value.

For each query, count the contribution from layers `0` and `1` separately using simple prefix sums. The remaining part projects onto a contiguous interval of layer `2`. Each projected element contributes according to how many cells of the rectangle lie on its diagonal; these multiplicities form an increasing arithmetic part, then a constant part, then a decreasing arithmetic part. Using the two prefix sums, each part is evaluated in `O(1)`.

Total complexity: `O(N + Q)` time and `O(N)` memory.
