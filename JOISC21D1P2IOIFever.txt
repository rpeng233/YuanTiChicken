# JOISC 2021 Day 1 Problem 2 — IOI Fever

## Key Idea

Fix the direction of citizen (1). There are only four possibilities, so solve each case separately and take the maximum.

Rotate the plane so that citizen (1) moves east from the origin.

For any other citizen, the Manhattan-distance bound implies that only one movement direction can possibly allow them to become infected:

* If (|x|>|y|), move horizontally toward the (y)-axis.
* If (|x|<|y|), move vertically toward the (x)-axis.
* The boundary case (|x|=|y|) is handled according to the direction of citizen (1).

Thus, after fixing citizen (1)'s direction, all citizens' useful directions are determined.

## Infection as Dijkstra

For two citizens (i,j), let (m(i,j)) be the time when they meet, or infinity if they never meet.

If citizen (i) becomes infected at time (d_i), then they can infect (j) only when

[
m(i,j)\ge d_i.
]

The earliest infection times satisfy a Dijkstra-like process:

[
d_1=0,
\qquad
d_j=\min_{i:,m(i,j)\ge d_i} m(i,j).
]

The problem is that explicitly considering every pair of citizens would take (O(N^2)).

## Compressing Possible Meetings

Two moving citizens can meet only when their initial positions lie on one of these line types:

[
x=\text{constant},\qquad
y=\text{constant},\qquad
x-y=\text{constant},\qquad
x+y=\text{constant}.
]

For each line type:

1. Group citizens lying on the same line.
2. Separate them according to movement direction.
3. Sort them by their position along the line.

For a fixed infected citizen, all citizens they can meet on one such line form a prefix or suffix of the sorted order. Their meeting times are monotone.

Therefore, instead of adding edges to every reachable citizen, add only an edge to the first eligible citizen. When that citizen is processed, continue to the next citizen in the sorted order.

This produces only (O(N)) auxiliary states and transitions.

## Algorithm

For each of the four possible directions of citizen (1):

1. Rotate coordinates so citizen (1) moves east.
2. Determine the unique useful direction of every citizen.
3. Build sorted lists for the four line types.
4. Run Dijkstra:

   * When a citizen is infected, binary-search each compatible line list for the first meeting occurring no earlier than their infection time.
   * Insert that meeting into the priority queue.
   * Processing a meeting also inserts the next meeting along the same sorted list.
5. Count the infected citizens.

Return the maximum count over the four orientations.

## Correctness

* The Manhattan-distance bound shows that, once citizen (1)'s direction is fixed, every infectable citizen must use the chosen useful direction.
* Dijkstra processes possible infection events in chronological order, so each citizen receives their earliest possible infection time.
* Along every collision line, meeting times are monotone, so generating the first eligible meeting and then advancing sequentially finds every relevant meeting.
* Trying all four directions of citizen (1) covers every possible optimal solution.

## Complexity

For each orientation, sorting and Dijkstra take

[
O(N\log N)
]

time and

[
O(N)
]

memory.

Since there are only four orientations, the total complexity remains

[
\boxed{O(N\log N)}
]

time and

[
\boxed{O(N)}
]

memory.
