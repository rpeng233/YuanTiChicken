The key transformation is to view the fixed labeled nodes as an unchanged virtual-tree skeleton. Extra nodes are generated by processing labels in increasing order; the actual shape of the given skeleton is irrelevant, only the current number of vertices and edges matters.

Maintain a bitmask DP over at most `k` pending placeholders. A placeholder means: when processing an earlier label, we inserted an unlabeled future branching ancestor on some existing edge, and it must later be filled by one of the next `k` labels.

Let `f[mask]` be the number of ways before processing the next label. If `i` new labels have already been processed, the current partial structure has

`cur = n + i + popcount(mask)`

vertices, hence `cur - 1` edges.

For each state:

- The current label fills one pending placeholder:  
  for every set bit `b` in `mask`, remove it and shift the mask to age all remaining placeholders.

- The current label is inserted normally, either as a leaf or on an edge:  
  `2 * cur - 1` choices.

- The current label creates a new pending parent: choose an existing edge, insert the placeholder there, and attach the current label below it:  
  `cur - 1` choices, then add a fresh pending bit.

In code form:

```text
f[0] = 1
for i = 0 .. m-1:
    clear g
    for mask:
        cur = n + i + popcount(mask)

        for each set bit b in mask:
            nm = (mask without b) << 1
            if nm < 2^k:
                g[nm] += f[mask]

        nm = mask << 1
        if nm < 2^k:
            g[nm] += f[mask] * (2 * cur - 1)
            if k > 0:
                g[nm | 1] += f[mask] * (cur - 1)

    f = g mod MOD
answer = f[0]
```

States where a placeholder survives for more than `k` steps are discarded by the mask overflow check. The final mask must be empty. Complexity is `O(m k 2^k)` time and `O(2^k)` memory.
